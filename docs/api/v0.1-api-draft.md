# v0.1 API Draft

This page defines the minimum provisional public surface for the first workspace bootstrap.

The goal is to give the repository a stable implementation target without implying that export fidelity or layout behavior already exists.

## Status

All APIs in this document are provisional.

They may change before the first preview release if the canonical IR or package boundaries require it.

## `@pptkit/core`

Planned minimum surface:

```ts
interface PresentationElement {
  type: string;
}

interface PresentationSlideInput {
  id?: string;
  elements?: PresentationElement[];
}

interface PresentationInit {
  title?: string;
}

interface PresentationSlide {
  id: string;
  elements: PresentationElement[];
}

interface PresentationDocument {
  title?: string;
  slides: PresentationSlide[];
  addSlide(input?: PresentationSlideInput): PresentationSlide;
}

declare function createPresentation(init?: PresentationInit): PresentationDocument;
```

Bootstrap expectations:

- `createPresentation()` returns an in-memory document shell.
- `addSlide()` appends a new slide with a generated identifier when one is not provided.
- Element typing remains intentionally broad until the canonical IR is defined.

Out of scope for `v0.1` bootstrap:

- full theme model
- rich text model
- image asset lifecycle
- tables, notes, animations, or master slides

## `@pptkit/layout`

Planned minimum surface:

```ts
interface LayoutResult {
  slideCount: number;
  status: "placeholder";
}

declare function resolveLayout(document: PresentationDocument): LayoutResult;
```

Bootstrap expectations:

- layout is a named package boundary, even before the real engine exists
- output stays obviously non-final

## `@pptkit/pptx-exporter`

Planned minimum surface:

```ts
interface ExportPptxOptions {
  output: string;
}

interface ExportAttempt {
  output: string;
  slideCount: number;
  status: "not-implemented";
}

declare function exportPptx(
  document: PresentationDocument,
  options: ExportPptxOptions,
): Promise<ExportAttempt>;
```

Bootstrap expectations:

- exporter accepts a core document and export options
- exporter is allowed to return a structured placeholder result during bootstrap
- the return shape should make it impossible to confuse bootstrap behavior with real file generation

## `@pptkit/cli`

Planned minimum surface:

```ts
interface CliRunResult {
  exitCode: number;
  output: string;
}

declare function runCli(args?: string[]): CliRunResult;
```

Bootstrap expectations:

- the CLI proves command wiring and packaging shape
- no file-based commands are required yet

## Review Triggers

Update this draft when any of the following happens:

- `PresentationElement` becomes a typed union
- the canonical IR requires different authoring entry points
- the exporter begins writing real `.pptx` output
- the CLI needs subcommands or configuration loading
